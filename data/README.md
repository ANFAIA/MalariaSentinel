# Data

Datasets used across the MalariaSentinel research.

## ⚠️ Gitignored — restore on demand

The raw DwC occurrence files (`.txt`) and DwC-A archive copies (`.zip`) are
**kept out of git** to avoid bloating the repo (~80 MB of text + archives). The
folder structure and metadata (`eml.xml`, `meta.xml`, `permit.txt`) are tracked,
but the bulk data is restored locally via:

```bash
uv run --with requests --with rasterio --with matplotlib --with numpy \
    python scripts/restore_data.py
```

See `scripts/restore_data.py` for the canonical recovery flow (datasets, SRTM
terrain, WorldClim/JRC env layers, and the Moua 2016 paper). The script is
idempotent — safe to re-run any time.

To restore only one dataset: `restore_data.py guf` (or `react`, `ghana_idit`,
`colombia_vl`).

## Notes

- The IPT archives occasionally change (columns get added/dropped). The GUF
  archive currently ships many DwC columns as NaN (sex, month, day, habitat,
  eventRemarks, waterBody, country, coordinatePrecision, typeStatus) — this is
  a difference in the upstream archive, not a restore bug. The
  `mal-data-explorer` scripts are defensive against empty fields.
- `data/colombia/colombia.zip` (a broken HTML file from an earlier failed
  attempt at a different Colombia dataset) is intentionally NOT restored.
